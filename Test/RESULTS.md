## Что работает ✅

1. **Фрагментация через один FFmpeg процесс** - все тайлы создаются за один проход
2. **Синхронизация** - все тайлы имеют одинаковую длительность (2.166s)
3. **Анимация сохранена** - все тайлы анимированные
4. **Оптимизация VP9** - используются параметры:
   - `-deadline realtime`
   - `-cpu-used 6`
   - `-row-mt 1`
   - `-crf 40`
5. **Контроль длительности** - setpts применяется до фрагментации
6. **Размер файлов** - с CRF=40 размер ~700-800KB (можно уменьшить до <512KB с CRF=45-50)

## Проблемы ⚠️

### 1. Прозрачность не сохраняется
**Причина**: GIF decoder в FFmpeg не корректно обрабатывает альфа-канал при конвертации в VP9
**Решение**: 
- Использовать WebP с прозрачностью на входе (вместо GIF)
- Или PNG последовательность
- Добавлен параметр `-auto-alt-ref 0` для поддержки альфы

### 2. Размер файлов >512KB
**Текущий**: 700-800KB с CRF=40
**Целевой**: <512KB
**Решение**: Увеличить CRF до 45-50 или реализовать binary search

## Архитектурные решения

### 1. Один FFmpeg процесс для всех тайлов ✅
```typescript
filter_complex: "
[0:v]fps=30,setpts=factor*PTS,split=4[v0][v1][v2][v3];
[v0]crop=512:512:0:0[t0];
[v1]crop=512:512:512:0[t1];
[v2]crop=512:512:0:512[t2];
[v3]crop=512:512:512:512[t3]
"
```

### 2. Оптимальные параметры VP9 ✅
- `libvpx-vp9` codec
- `yuva420p` pixel format (для альфы)
- `-auto-alt-ref 0` (обязательно для альфы)
- `-deadline realtime` (скорость)
- `-cpu-used 6` (баланс скорость/качество)
- `-row-mt 1` (многопоточность)
- `-crf 40-50` (контроль размера)
- `-b:v 0` (VBR режим)
- `-an` (без аудио)

### 3. Порядок операций ✅
```
GIF → scale → fps + setpts → split → crop → VP9 encode
```

## Рекомендации

1. **Для production**: использовать WebP на входе вместо GIF
2. **Binary search CRF**: реализовать для точного контроля размера <512KB
3. **Тестирование с PNG**: проверить сохранение прозрачности
4. **Метрики**: добавить логирование времени выполнения каждого этапа
