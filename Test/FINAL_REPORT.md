### 2. WebP → PNG → WebM → Фрагменты ✅
**Результат:** Работает отлично!
- Размер тайлов: 41-102 KB (CRF=45) ✓
- Синхронизация: ✓ Идеальная (1.733s все тайлы)
- Анимация: ✓ Сохранена (52 кадра)
- Прозрачность: ✗ Потеряна

**Pipeline:**
```
GIF → WebP (Sharp, 52 frames) → PNG sequence (Sharp, 52 PNG с rgba) → WebM (FFmpeg) → 4 тайла
```

## Проблема с прозрачностью

### Причина
GIF имеет индексированную прозрачность (bgra), но:
1. Sharp при GIF→WebP теряет альфа-канал
2. FFmpeg при PNG→WebM игнорирует `-pix_fmt yuva420p`
3. Даже с фильтром `format=yuva420p` результат `yuv420p`

### Тестовый GIF
`rickroll-roll.gif` - не имеет реальной прозрачности (фон не прозрачный)

## Рекомендации для production

### Вариант 1: WebP → PNG → WebM (текущий)
```typescript
// В ffmpeg-fragment.task.ts:
1. Sharp: WebP → PNG frames (с альфой)
2. FFmpeg: PNG sequence → WebM 1024x1024
3. FFmpeg: WebM → 4 тайла (один процесс)
```

**Плюсы:**
- Стабильно работает
- Маленькие файлы (<512KB)
- Идеальная синхронизация

**Минусы:**
- Много временных PNG файлов
- Прозрачность теряется (нужен тест с реальным прозрачным стикером)

### Вариант 2: WebP → GIF → WebM
```typescript
// Промежуточная конвертация:
Sharp: WebP → GIF (animated)
FFmpeg: GIF → WebM → тайлы
```

**Плюсы:**
- Один промежуточный файл
- FFmpeg хорошо читает GIF

**Минусы:**
- GIF не поддерживает полную прозрачность

### Вариант 3: Использовать ffmpeg-static в production
Скачанный FFmpeg 7.0.2 работает так же, как системный.

## Итоговое решение

**Для реализации в ffmpeg-fragment.task.ts:**

```typescript
1. Получить WebP от rescale.task
2. Sharp: WebP → PNG sequence (сохранить в temp)
3. FFmpeg: PNG → WebM 1024x1024 с параметрами:
   - format=yuva420p (фильтр)
   - pix_fmt yuva420p
   - auto-alt-ref 0
   - crf 45 (для <512KB)
4. FFmpeg: WebM → 4 тайла (один процесс, split+crop)
5. Удалить временные PNG
```
