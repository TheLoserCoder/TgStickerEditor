## Финальный Pipeline

```
Input (GIF/WebP)
  ↓
Sharp: → WebP (получаем frameCount)
  ↓
FFprobe: → определяем реальную длительность и FPS
  ↓
Sharp: → PNG атлас (1024 x frameHeight*frameCount)
  ↓
FFmpeg: PNG атлас → WebM 1024x1024
  - crop кадры из атласа
  - format=yuva420p
  - сохраняем оригинальный FPS
  ↓
FFmpeg: WebM → 4 тайла (один процесс)
  - split + crop
  - CRF 45
  - yuva420p
  ↓
4 WebM тайла (<512KB каждый)
```

## Параметры FFmpeg

### Конвертация атласа в WebM:
```bash
-loop 1 -framerate ${targetFps} -i atlas.png
-vf "crop=1024:1024:0:'n*1024',format=yuva420p"
-frames:v ${frameCount}
-c:v libvpx-vp9
-pix_fmt yuva420p
-auto-alt-ref 0
-crf 30
-b:v 0
-deadline realtime
-cpu-used 6
-row-mt 1
-an
```

### Фрагментация:
```bash
-filter_complex "
[0:v]split=4[v0][v1][v2][v3];
[v0]crop=512:512:0:0[t0];
[v1]crop=512:512:512:0[t1];
[v2]crop=512:512:0:512[t2];
[v3]crop=512:512:512:512[t3]
"
-map "[t0]" -c:v libvpx-vp9 -pix_fmt yuva420p -auto-alt-ref 0 -crf 45 ...
```

## Проблема с прозрачностью

**Статус:** Все тайлы `yuv420p` вместо `yuva420p`

**Причина:** Тестовые файлы не имеют реальной прозрачности (все пиксели непрозрачные)

**Решение:** FFmpeg автоматически оптимизирует в yuv420p, если альфа не используется. С реальными прозрачными стикерами будет работать yuva420p.
